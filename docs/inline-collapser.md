## **InlineCollapser**

---

### 1. Назначение

`InlineCollapser` — сервис, который объединяет последовательность строчных (inline) узлов в один или несколько элементов с единым текстовым содержимым и массивом сущностей форматирования. Это позволяет преобразовать мелкие фрагменты (например, `<strong>`, `<em>`, текстовые узлы) в читаемый текст с сохранением информации о форматировании.

---

### 2. Принцип работы

1. На вход подаётся коллекция `ElementList` (дочерние элементы текущего узла).
2. Проверяется, что все элементы являются inline (метод `isInline()` возвращает `true`). Если нет — схлопывание не применяется.
3. Выполняется последовательный обход элементов:
   - Текстовые узлы (`TextElement`) добавляют свой текст.
   - Элементы форматирования (`strong`, `em`, `a` и т.п.) добавляют текст из своего `label` или `data`, а также создают сущность с типом, позициями и атрибутами.
4. В результате формируется один (или несколько) новых элементов:
   - Обычно создаётся один элемент того же типа, что и родительский контекст (например, `div`), но с объединённым текстом в `data`/`label` и массивом сущностей в `entities`.
   - В некоторых случаях (например, при наличии блоков внутри) схлопывание может породить несколько элементов.

**Выход:** новый `ElementList`, содержащий один или более элементов.

---

### 3. Сущности форматирования

Сущность — это ассоциативный массив с информацией о форматировании фрагмента текста:

```php
[
    'type'       => 'bold',      // тип: bold, italic, link, span и т.д.
    'start'      => 5,           // начальная позиция в объединённом тексте (байты или символы)
    'end'        => 10,          // конечная позиция
    'attributes' => [            // дополнительные атрибуты (например, href для ссылки)
        'href' => 'https://...'
    ]
]
```

Позиции отсчитываются от начала объединённого текста. Вопрос о байтах/символах решается в зависимости от настроек (рекомендуется использовать символы через `mb_*` функции).

---

### 4. Интерфейс InlineCollapser

**Пространство имён:** `HtmlDomParser\Core\InlineCollapser`  
Реализует `InlineCollapserInterface`.

**Методы:**

| Метод | Описание |
|-------|----------|
| `collapse(ElementList $children, array $options = []): ElementList` | Выполняет схлопывание. Возвращает новую коллекцию. |
| `setParentContext(?NodeContext $context): void` | Устанавливает контекст родителя (может влиять на поведение). |

**Опции:**
- `preserveWhitespace` – сохранять ли пробелы между элементами (по умолчанию `true`).
- `encoding` – кодировка для расчёта позиций (по умолчанию `UTF-8`).

---

### 5. Алгоритм (псевдокод)

```
функция collapse(children):
    если не все дети inline:
        вернуть children (без изменений)
    текст = ""
    сущности = []
    смещение = 0
    для каждого child в children:
        если child это TextElement:
            добавить child.data к тексту
            смещение += длина(child.data)
        иначе:
            // child — элемент форматирования (strong, em, a)
            текст_ребенка = child.getText()  // обычно child.label или child.data
            начало = смещение
            добавить текст_ребенка к тексту
            смещение += длина(текст_ребенка)
            конец = смещение
            сущности[] = [
                'type' => child.name,
                'start' => начало,
                'end' => конец,
                'attributes' => child.attributes
            ]
    // Создаём новый элемент (например, тот же тип, что у родителя)
    новый_элемент = new Element(parentContext.getName())
    новый_элемент.setData(текст)
    новый_элемент.setEntities(сущности)
    вернуть новый ElementList([новый_элемент])
```

---

### 6. Пример

Исходный HTML:
```html
<p>Hello <strong>world</strong>!</p>
```

До схлопывания (в контексте `<p>`):
- `children` содержит:
  - `TextElement` с `data="Hello "`
  - `Element` (strong) с `label="world"`, `name="strong"`
  - `TextElement` с `data="!"`

После `collapse`:
- Новый `Element` (p) с `data="Hello world!"`
- `entities` = [
    ['type' => 'strong', 'start' => 6, 'end' => 11, 'attributes' => []]
  ]

---

### 7. Интеграция с событиями

- Перед схлопыванием вызывается событие `pre-inline-collapse`, которому передаётся текущий контекст. Обработчик может модифицировать список детей или отменить схлопывание.
- После схлопывания вызывается `post-inline-collapse` с обновлённым контекстом (уже с заменёнными детьми).
