# Введение в HtmlDomParser

**HtmlDomParser** — это PHP-библиотека для продвинутого парсинга HTML, которая преобразует исходный код в удобное объектное дерево с богатой семантической информацией. Это не просто очередной парсер, а полноценный фреймворк для построения семантически-обогащенного DOM-дерева.

## Что делает библиотеку особенной?

В отличие от стандартных парсеров, которые возвращают сырое DOM-дерево, HtmlDomParser выполняет рекурсивный обход с **контекстной обработкой** каждого узла:

```
HTML-строка 
    → Загрузка в DOMDocument 
    → Рекурсивный обход с созданием контекста для каждого узла 
    → Обогащение семантической информацией 
    → Итоговое дерево (Document с элементами)
```

## Ключевые возможности

### 🧠 Контекстная обработка узлов

Для каждого DOM-узла создается специальный объект **контекста**, который хранит:
- Ссылку на оригинальный узел
- Правила обработки тега (строчный, блочный, пустой)
- Информацию о разрешенных дочерних тегах
- Результаты промежуточных вычислений

Правила берутся из встроенной **карты контекстов тегов**, содержащей информацию для всех стандартных HTML-тегов.

### 🔗 Схлопывание строчного контента

Одна из ключевых особенностей — объединение последовательности строчных элементов (текст, выделения, ссылки) в один элемент с единым текстом и массивом **сущностей форматирования**:

```php
// HTML: <p>Это <b>жирный</b> и <i>курсивный</i> текст</p>

// После парсинга:

$element->getLabel();
// "Это жирный и курсивный текст"

$element->getEntities(); 
// [
//   ['type' => 'b', 'start' => 4, 'end' => 10],
//   ['type' => 'i', 'start' => 13, 'end' => 22]
// ]
```

### 🧩 Расширяемость через события и модули

- **Событийная модель** — в ключевых точках обработки (до/после обработки детей, до/после схлопывания) генерируются события. Любой желающий может подписаться на них и внедрить свою логику.
- **Модульная архитектура** — модули оформляются как Composer-пакеты, автоматически обнаруживаются и могут предоставлять свой API для работы с готовым деревом (например, модуль для поиска по CSS-селекторам).

### ⚠️ Продвинутая обработка ошибок

Все ошибки парсинга централизованно собираются в обработчике ошибок и классифицируются по уровням серьезности:
- **Уведомления (notice)** — не влияют на результат
- **Предупреждения (warning)** — возможны незначительные проблемы
- **Фатальные ошибки (error)** — результат может быть некорректен

Для каждой проблемы создается специальный **узел-ошибка**, замещающий проблемный HTML-узел в дереве. Поведение для каждого уровня можно настроить: либо просто собирать ошибки, либо выбрасывать исключение.

## Итоговая модель данных

Результатом работы является дерево, состоящее из объектов:

- **Узел (Node)** — базовая единица с именем и атрибутами
- **Элемент (Element)** — основной тип узла для тегов, содержит:
  - **Data** — основное смысловое содержимое (URL для ссылки, src для изображения)
  - **Label** — текстовая метка (например, текст ссылки)
  - **Entities** — массив сущностей форматирования после схлопывания
  - **Children** — коллекция дочерних элементов
- **Документ (Document)** — корневой объект, содержащий всё дерево и ссылку на обработчик ошибок
