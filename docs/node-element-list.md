## Node, Element, ElementList

### Введение

В результате парсинга HTML-дерево преобразуется в иерархию объектов-узлов. Все узлы наследуют базовый класс `Node`. Элементы (`Element`) представляют HTML-теги и могут содержать дочерние узлы. Для хранения дочерних элементов используется коллекция `ElementList`.

---

### 1. Класс Node (базовый)

**Пространство имён:** `HtmlDomParser\Core\Node\Node`

`Node` – базовый класс для всех узлов дерева. Не является абстрактным, но обычно создаются его наследники.

#### Свойства (все объявлены как private/protected)

| Свойство | Тип | Описание |
|----------|-----|----------|
| `$name` | `string` | Имя узла (например, `'div'`, `'#text'`, `'#comment'`). |
| `$attributes` | `array` | Ассоциативный массив атрибутов (для элементов). Для текстовых узлов и комментариев пуст. |

#### Методы

| Метод | Описание |
|-------|----------|
| `getName(): string` | Возвращает имя узла. |
| `getAttributes(): array` | Возвращает все атрибуты. |
| `getAttribute(string $name, $default = null): mixed` | Возвращает значение атрибута или `$default`. |
| `hasAttribute(string $name): bool` | Проверяет наличие атрибута. |
| `setAttribute(string $name, $value): void` | Устанавливает атрибут. |
| `removeAttribute(string $name): void` | Удаляет атрибут. |

---

### 2. Класс Element

**Пространство имён:** `HtmlDomParser\Core\Node\Element`

`Element` представляет HTML-элемент. Наследует `Node`. Может содержать дочерние узлы, хранящиеся в объекте `ElementList`.

#### Свойства (все объявлены как private/protected)

| Свойство | Тип | Описание |
|----------|-----|----------|
| `$data` | `mixed` | Основное содержимое, извлечённое `DataResolver` (обычно строка или `null`). |
| `$label` | `string` | Текстовая метка (для ссылок хранит текст ссылки, для остальных может дублировать `data`). |
| `$entities` | `array` | Массив сущностей форматирования после схлопывания. |
| `$isRawText` | `bool` | Флаг «сырого» текста (`true` для `script`, `style`). |
| `$isEscapable` | `bool` | Всегда `true` (преобразование сущностей). |
| `$children` | `ElementList` | Коллекция дочерних элементов (после схлопывания может отсутствовать). |

#### Методы для работы с данными

| Метод | Описание |
|-------|----------|
| `getData(): mixed` | Возвращает `data`. |
| `setData($value): void` | Устанавливает `data`. |
| `getLabel(): string` | Возвращает `label`. |
| `setLabel(string $label): void` | Устанавливает `label`. |
| `getEntities(): array` | Возвращает массив сущностей. |
| `addEntity(array $entity): void` | Добавляет сущность. |

#### Методы для схлопывания

| Метод | Описание |
|-------|----------|
| `isInline(): bool` | Возвращает `true`, если элемент относится к категории inline (определяется по правилам). |
| `shouldCollapse(): bool` | Возвращает `true` для inline-элементов, не являющихся raw-текстом. |
| `getText(): string` | Возвращает текстовое содержание элемента (с учётом детей или `data`/`label`). |
| `setText(string $text): void` | Устанавливает текстовое содержание (записывает в `data` или `label` в зависимости от типа). |

#### Методы для работы с потомками

| Метод | Описание |
|-------|----------|
| `getChildren(): ElementList` | Возвращает коллекцию дочерних элементов. |
| `hasChildren(): bool` | Проверяет наличие детей. |
| `addChild(Element $child): void` | Добавляет дочерний элемент. |
| `removeChild(Element $child): void` | Удаляет дочерний элемент. |

#### Методы для навигации

| Метод | Описание |
|-------|----------|
| `getParent(): ?Element` | Возвращает родительский элемент (если есть). |
| `setParent(?Element $parent): void` | Устанавливает родителя. |

#### Методы для экспорта

| Метод | Описание |
|-------|----------|
| `toHtml(): string` | Рекурсивно строит HTML-представление элемента. |
| `toArray(): array` | Преобразует элемент и его потомков в массив (для отладки, экспорта). |

### 3. Наследники Element

**Пространство имён:** `HtmlDomParser\Core\Node`

#### `VoidElement`

- Самозакрывающиеся теги (`img`, `br`, `input` и т.д.).
- Не могут содержать детей (`hasChildren()` всегда `false`).
- Не участвуют в схлопывании.

#### `TextElement`

- Представляет текстовый узел (`#text`).
- `isInline()` всегда `true`.
- `data` содержит текст после нормализации.
- `entities` всегда пуст.

#### `CommentElement`

- Представляет комментарий (`#comment`).
- Сохраняется только при `$keepComments = true` в конструкторе `Parser`.
- `data` содержит текст комментария.

#### `ErrorElement`

- Специализированный узел для ошибок (подробнее в документе **«ErrorHandler + ErrorElement»**).
- Содержит диагностическую информацию.

### 4. Класс Document

**Пространство имён:** `HtmlDomParser\Core\Node\Document`

`Document` – корневой узел документа. Наследует `Node`.

#### Свойства (все объявлены как private/protected)

| Свойство | Тип | Описание |
|----------|-----|----------|
| `$rootElement` | `?Element` | Корневой элемент документа (обычно `html`). |
| `$errorHandler` | `ErrorHandler` | Ссылка на обработчик ошибок (для доступа к ошибкам после парсинга). |

#### Методы

| Метод | Описание |
|-------|----------|
| `getRootElement(): ?Element` | Возвращает корневой элемент. |
| `getErrorHandler(): ErrorHandler` | Возвращает обработчик ошибок. |
| `toArray(): array` | Рекурсивно экспортирует всё дерево в массив. |
| `toHtml(): string` | Строит полный HTML-код документа. |

---

### 5. Класс ElementList

**Пространство имён:** `HtmlDomParser\Core\Collection\ElementList`

`ElementList` – коллекция объектов `Element`, реализующая интерфейсы `IteratorAggregate` и `Countable`. Используется для хранения дочерних узлов в `Element` и контексте.

#### Основные методы

| Метод | Описание |
|-------|----------|
| `push(Element $element): void` | Добавляет элемент в конец списка. |
| `get(int $index): ?Element` | Возвращает элемент по индексу. |
| `count(): int` | Возвращает количество элементов. |
| `getIterator(): ArrayIterator` | Возвращает итератор для перебора. |
| `toArray(): array` | Преобразует коллекцию в массив элементов. |
| `filter(callable $callback): ElementList` | Возвращает новую коллекцию, отфильтрованную по callback. |
| `map(callable $callback): array` | Применяет callback к каждому элементу и возвращает массив результатов. |

---

### 6. Примеры использования

```php
// Получение корневого элемента
$root = $document->getRootElement();

// Обход детей
foreach ($root->getChildren() as $child) {
    echo $child->getName() . ': ' . $child->getText() . "\n";
}

// Поиск элемента по тегу (пример с фильтрацией)
$divs = $root->getChildren()->filter(fn($el) => $el->getName() === 'div');

// Доступ к данным
$link = $divs[0]->getChildren()[0];
echo $link->getData();   // href
echo $link->getLabel();  // текст ссылки
```
