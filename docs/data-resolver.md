## DataResolver

### Назначение
`DataResolver` — компонент ядра, отвечающий за определение основного содержимого (`data`) HTML-элемента на основе его типа и атрибутов. Он преобразует сырой DOM-узел в строковое значение, которое затем сохраняется в поле `data` соответствующего объекта `Element`.

### Расположение
- **Класс**: `HtmlDomParser\Core\DataResolver`
- **Интерфейс**: `HtmlDomParser\Contract\DataResolverInterface`

### Принцип работы
1. На вход поступает объект `\DOMNode`.
2. По имени узла (`nodeName`) определяется правило (источник данных) из внутренней таблицы.
3. В зависимости от правила извлекается значение:
   - если правило — название атрибута (например, `href`), возвращается значение этого атрибута (при его наличии);
   - если правило — `#text`, возвращается текстовое содержимое узла (`nodeValue` или `textContent`);
   - если правило отсутствует или атрибут не найден, возвращается `null`.

### Таблица источников `data`

| Источник | Теги |
|----------|------|
| `href` | a, area, base, link |
| `src` | audio, embed, iframe, img, input (image), script, source, track, video |
| `value` | button, data, input, li, option, param, meter |
| `action` | form |
| `content` | meta |
| `data` | object |
| `datetime` | time |
| `#text` | script (если нет src), style, textarea, title, option (если нет value), а также все текстовые узлы (`#text`) |
| `null` | все остальные теги (например, div, p, span) |

Таблица правил является фиксированной и не предназначена для изменения пользователем. При необходимости нестандартного извлечения данных рекомендуется использовать события (например, pre-node) или создавать модули.

### Использование в парсере
`DataResolver` создаётся один раз в конструкторе `Parser` и передаётся в `NodeContext`. Контекст обращается к резолверу лениво — при первом вызове `getData()`. Результат кэшируется в контексте, что позволяет избежать повторных вызовов для одного узла.

```php
$dataResolver = new DataResolver();
$data = $dataResolver->resolve($domNode); // ?string
```

### Взаимодействие с `label` и `entities`
- Для ссылок (`<a>`) `data` хранит атрибут `href`, а текст ссылки сохраняется в поле `label` элемента (заполняется позже при схлопывании).
- Для остальных элементов `label` по умолчанию дублирует `data` или остаётся пустым.
- Сущности форматирования (`entities`) формируются независимо, в процессе схлопывания, и не влияют на `data`.

### Методы

| Метод | Описание |
|-------|----------|
| `resolve(\DOMNode $node): ?string` | Возвращает основное содержимое узла или `null`. |

**Примечание:** Дополнительные методы `setRule()`, `getRule()` и `getRules()` предназначены для внутреннего использования и не документируются как публичный API.

### Исключения и ошибки
- При невозможности определить `data` (отсутствие обязательного атрибута) резолвер не генерирует исключение, а возвращает `null`. Соответствующая ошибка (уровня `NOTICE` или `WARNING`) создаётся позже в `NodeContext` или `ErrorHandler`.
- Валидация входных данных не производится — предполагается, что передан корректный объект `DOMNode`.

### Примеры

```php
$resolver = new DataResolver();

$dom = new \DOMDocument();
$dom->loadHTML('<a href="https://example.com">link</a>');
$linkNode = $dom->getElementsByTagName('a')->item(0);

echo $resolver->resolve($linkNode); // 'https://example.com'

$textNode = $dom->createTextNode('plain text');
echo $resolver->resolve($textNode); // 'plain text'
```
